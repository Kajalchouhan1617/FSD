TOMATO FOOD DELIVERY SYSTEM

Interactive Project Report

Abstract

This report details the development of "Tomato Food Delivery," a comprehensive food ordering and delivery system built using the MERN (MongoDB, Express.js, React, Node.js) stack. The project aims to provide a seamless and efficient platform connecting customers with local restaurants for food ordering and delivery. The system incorporates features such as user registration and authentication, restaurant listing, menu browsing, order placement, real-time order tracking, and secure payment processing. For restaurants, it offers functionalities for menu management, order reception, and status updates. The MERN stack was chosen for its robust, scalable, and full-stack JavaScript capabilities, enabling rapid development and a highly interactive user experience. This document outlines the problem statement, system requirements, architectural design, implementation details, testing procedures, and future enhancements of the "Tomato Food Delivery" application. The project addresses the growing demand for convenient food delivery services by offering a user-friendly and reliable solution.

Chapter 2: Introduction to Problem

This chapter delves into the problem that the "Tomato Food Delivery" system aims to solve. 2.1 Overview

In today's fast-paced world, convenience plays a significant role in daily life. Food delivery services have become an indispensable part of this convenience, allowing individuals to order meals from their favorite restaurants and have them delivered to their doorstep. The demand for such services has surged, driven by busy lifestyles, technological advancements, and the desire for diverse culinary experiences without the need to dine out. However, many existing platforms often face challenges such as complex user interfaces, limited restaurant choices, inefficient order tracking, and occasional delivery delays. The "Tomato Food Delivery" project aims to address these pain points by providing a streamlined, intuitive, and reliable platform that enhances the overall food ordering and delivery experience.

2.2 Existing System

The current landscape of food delivery services is dominated by several large players, each offering a range of features. These systems typically allow users to browse menus, place orders, and track deliveries. Restaurants use these platforms to manage their online presence, update menus, and process incoming orders. While these systems have revolutionized food access, they often present certain limitations. Some platforms may have high commission rates for restaurants, leading to increased food prices for consumers. Others might suffer from a lack of personalization, generic user interfaces, or a fragmented experience across different devices. Furthermore, real-time tracking and efficient customer support can sometimes be lacking, leading to user frustration. The "Tomato Food Delivery" system seeks to learn from these existing models and improve upon them by focusing on a user-centric design, robust backend, and efficient logistics.

4.1 Introduction to Languages (Front End and Back End)

Backend (Node.js & Express.js): Node.js provides the JavaScript runtime for server-side logic. Express.js, a minimal framework for Node.js, is used to build robust RESTful APIs for user authentication, data handling, and order processing.

Frontend (React): React, a JavaScript library, is used for building the interactive user interface. Its component-based architecture allows for modular and reusable UI elements, enhancing the user experience.

Database (MongoDB): MongoDB, a NoSQL document database, stores data in flexible JSON-like documents, suitable for evolving application requirements and scaling.

4.3 Implementation of Problem

This section provides a glimpse into the implementation logic, including pseudo-code for key functionalities like user authentication and order placement, and a table of primary API endpoints. It illustrates how different parts of the system interact.

User Authentication (Backend Pseudo-code)


// User Registration
FUNCTION registerUser(username, email, password)
  IF email or username already exists THEN
    RETURN Error: "User already exists"
  END IF
  HASH password
  CREATE new User document with hashed password
  SAVE User document to MongoDB
  GENERATE JWT token
  RETURN Success: User and token
END FUNCTION

// User Login
FUNCTION loginUser(email, password)
  FIND User by email in MongoDB
  IF User not found THEN
    RETURN Error: "Invalid credentials"
  END IF
  COMPARE password with hashed password
  IF passwords do not match THEN
    RETURN Error: "Invalid credentials"
  END IF
  GENERATE JWT token
  RETURN Success: User and token
END FUNCTION

Order Placement (High-level Algorithm)

  1. Frontend (React): User adds items to cart, proceeds to checkout, provides details, and submits order. POST request to `/api/orders`.
  2. Backend (Node.js/Express.js): Receives order, validates data, processes payment, creates `Order` document in MongoDB (status: "pending"), notifies restaurant.
  3. Database (MongoDB): Stores the new `Order` document.

Key API Endpoints (Table 4.1)
